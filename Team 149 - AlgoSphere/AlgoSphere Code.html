Pick the Right Tool

Understand growth rates, compare data structures, and know when each algorithm shines.

O(1)
Constant
Does not grow with input size. Always the same amount of work regardless of n.
Array[i], HashMap get
O(log n)
Logarithmic
Halves the problem each step. n=1,000,000 only needs ~20 steps.
Binary Search, BST ops
O(n)
Linear
Work grows proportionally with input. Doubling n doubles the work.
Linear Search, Array scan
O(n log n)
Linearithmic
Slightly worse than linear — the sweet spot for comparison-based sorting.
Merge Sort, Quick Sort
O(n²)
Quadratic
Nested loops. Doubling n quadruples the work. Avoid for large inputs.
Bubble Sort, nested loops
